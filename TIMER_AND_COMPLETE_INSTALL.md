## 🔍 Как работает:

### Автоматическое завершение:
1. При каждом открытии страницы решения (`GET /api/decisions/{id}`)
2. Бэкенд проверяет: `decision.Deadline <= DateTime.UtcNow`
3. Если да и голосование ещё активно → автоматически завершает
4. Меняет `IsCompleted = true` и `Status = Completed`

### Ручное завершение:
1. Проверка на фронте: пользователь = создатель группы
2. Кнопка видна только создателю
3. При нажатии → подтверждение (confirm dialog)
4. Запрос `PUT /api/decisions/{id}/complete`
5. Бэкенд проверяет права (только создатель группы)
6. Завершает голосование и возвращает успех

### Таймер:
1. JavaScript `setInterval` каждую секунду
2. Вычисляет разницу: `deadline - now`
3. Форматирует как "2д 3ч 15м" или "45м 12с"
4. При истечении времени: показывает "Время истекло" и перезагружает страницу

## ⚠️ Важные моменты:

1. **Время сервера vs клиента:**
   - Дедлайн хранится в UTC в БД
   - Проверка на бэкенде использует `DateTime.UtcNow`
   - На фронте JavaScript автоматически конвертирует в локальное время

2. **Обновление страницы:**
   - При истечении таймера страница автоматически обновляется через 2 секунды
   - При ручном завершении — через 1.5 секунды
   - Это нужно, чтобы получить актуальный статус с сервера

3. **Проверка прав:**
   - Фронтенд делает дополнительный запрос `GET /api/groups/{id}` для проверки `creatorId`
   - Бэкенд также проверяет права при `PUT /api/decisions/{id}/complete`
   - Двойная проверка для безопасности

## 🐛 Возможные проблемы:

### "Кнопки завершения не видно"
- Убедись, что ты создатель группы (не просто участник)
- Проверь консоль браузера (F12) на ошибки
- Проверь, что `decision.groupId` возвращается с бэкенда

### "Таймер не обновляется"
- Проверь, что `decision.deadline` не null
- Убедись, что дедлайн в будущем
- Проверь консоль на ошибки JavaScript

### "Автоматическое завершение не работает"
- Проверь, что дедлайн прошёл
- Открой страницу решения (вызовется `GetDecision`)
- Проверь логи бэкенда: должно быть `[AUTO COMPLETE]`
